# Quickstart Guide: PoE Link Collection Hub Page

**Date**: 2025-12-04  
**Feature**: PoE Link Collection Hub Page

## Prerequisites

- Node.js 18+ installed
- Git installed
- Modern web browser (Chrome, Firefox, Edge, Safari)

## Initial Setup

### 1. Clone and Install

```bash
# Clone repository (if not already done)
git clone <repository-url>
cd poe-link-collection

# Install dependencies
npm install
```

### 2. Project Structure

```
poe-link-collection/
├── src/
│   ├── index.html          # Main HTML file
│   ├── styles/            # CSS files
│   ├── scripts/           # JavaScript files
│   └── data/              # JSON data files
├── tests/                 # Test files
├── public/                # Static assets
└── dist/                  # Build output (generated)
```

## Development

### Start Development Server

```bash
npm run dev
```

This starts Vite development server, typically at `http://localhost:5173`

### Development Workflow

1. Edit files in `src/` directory
2. Changes hot-reload automatically
3. Check browser console for errors
4. Test in multiple browsers

### Key Files to Edit

- **Links Data**: `src/data/links.json`
- **Events Data**: `src/data/events.json`
- **Styles**: `src/styles/main.css` and `src/styles/theme.css`
- **JavaScript**: `src/scripts/*.js`
- **HTML**: `src/index.html`

## Adding New Links

### Edit links.json

```json
{
  "builds": {
    "id": "builds",
    "title": "BUILDS",
    "links": [
      {
        "name": "New Resource",
        "url": "https://example.com",
        "icon": "optional-icon-id",
        "description": "Optional description"
      }
    ]
  }
}
```

### Adding New Category

Add a new top-level key in `links.json`:

```json
{
  "new-category": {
    "id": "new-category",
    "title": "NEW CATEGORY",
    "links": [...]
  }
}
```

## Adding Events

### Edit events.json

```json
[
  {
    "id": "new-league",
    "name": "New League Name",
    "startDate": "2025-01-01T16:00:00Z",
    "endDate": "2025-04-01T16:00:00Z",
    "type": "league"
  }
]
```

**Date Format**: Use ISO 8601 format (YYYY-MM-DDTHH:mm:ssZ)

## Deployment to GitHub Pages

### Automatic Deployment

1. Enable GitHub Pages in repository settings (Settings → Pages)
2. Set source to GitHub Actions
3. Push changes to `main` branch
4. GitHub Actions workflow automatically:
   - Runs tests
   - Builds the site
   - Deploys to GitHub Pages
5. Site available at your custom domain (if configured) or `https://<username>.github.io/<repo-name>/`

### Manual Deployment

```bash
# Build the site
npm run build

# Preview production build
npm run preview

# The dist/ folder contains the deployable site
# Upload dist/ contents to your hosting provider
```

## Common Tasks

### Adding a New Navigation Page

1. Create new HTML file in `src/` (e.g., `guide.html`)
2. Update navigation data in JavaScript
3. Add route handling in `navigation.js`

### Changing Theme Colors

Edit `src/styles/theme.css` CSS variables:

```css
:root {
  --poe-primary-color: #your-color;
  --poe-background: #your-bg-color;
}
```

### Debugging

- Check browser console for JavaScript errors
- Use browser DevTools to inspect elements

## Troubleshooting

### Links Not Loading

- Check `src/data/links.json` exists and is valid JSON
- Check browser console for fetch errors
- Verify file paths in JavaScript

### Events Not Displaying

- Check `src/data/events.json` exists and is valid JSON
- Verify date format is ISO 8601
- Check date calculation logic in `events.js`

### Build Errors

- Run `npm install` to ensure dependencies are installed
- Check Node.js version (requires 18+)
- Clear `node_modules` and reinstall if needed

### GitHub Pages Not Updating

- Verify GitHub Actions workflow is enabled
- Check Actions tab for build errors
- Ensure `dist/` directory is being deployed
- Verify repository settings for Pages deployment
